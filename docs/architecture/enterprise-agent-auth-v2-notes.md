## V2 已经成立的结论

- 企业需要的是面向 Skill 鉴权的轻量 IAM，而不是通用重型 IAM
- 鉴权链路必须同时包含 AuthCLI 前置拦截和服务端 verify
- 飞书是唯一主身份源
- 权限模型以 `user_id + skill_id + agent_id + chat_id + data_scope` 为核心
- 群聊权限必须和角色权限解耦，单独建模

## V2 到 V3 的主要差距

- V2 偏“方案叙述”，V3 要改成“实施蓝图”
- V2 把架构、接口、SQL、路线图、示例都堆在一个文档中，难以并行交付
